## User Flows

### Casual User Flow (滑鼠操作)
```
1. 點擊「銷貨發票」選單 → 2. 點擊「建立發票」
3. 點擊客戶下拉選單 → 4. 搜尋或選擇客戶
5. 點擊「+ 新增品項」→ 6. 填入品名、數量、單價
7. 選擇稅率 → 8. 點擊「送出eGUI」
9. 系統顯示「已送出」狀態 → 10. 可關閉視窗，系統 Tray 背景執行
```

### Power User Flow (鍵盤快捷)
```
1. Ctrl+Shift+N        → 新建發票（跳過選單）
2. Tab                  → 在欄位間移動
3. Ctrl+Enter          → 儲存草稿
4. Ctrl+Shift+Enter    → 送出eGUI
5. Ctrl+B               → 新增品項（+ 新增品項捷徑）
6. Esc                  → 取消/關閉
```

## Taiwan Localization Details

| 項目 | 顯示格式 | 說明 |
|------|----------|------|
| 發票日期 | ROC格式 `YYY/MM/DD` | 114/03/30 = 民國114年3月30日 |
| 稅率 | 下拉選單 | 5% (應稅), 0% (免稅), 內含 |
| 課稅別 | 下拉選單 | 應稅/免稅/特種稅率 |
| 品名 | 繁體中文 | 可依據庫存自動帶出 |
| 金額 | 千分位 + 小數點2位 | 計算時保持精度 |

## shadcn/ui Components

- **Table**: 發票品項明細，inline editable cells
- **Select**: 客戶選擇、稅率、課稅別
- **Input**: 品名（可搜尋）、數量、單價、備註
- **Button**: `primary` (送出eGUI), `secondary` (儲存草稿), `outline` (取消)
- **Badge**: eGUI狀態指示 (草稿=gray, 已送出=yellow, 已開立=green, 已作廢=red)
- **Dialog**: 確認送出、錯誤訊息
- **Toast**: 操作成功/失敗通知
- **Form**: Zod validation for all fields

## Tauri Integration Notes

- eGUI API 狀態查詢：後台定時 polling (60秒)，Tauri端處理
- 系統 Tray：送出後視窗可關閉，Tray icon 保持，通知送達時跳出系統通知
- 通知權限：首次送出前需 request macOS notification permission
- 草稿自動儲存：每30秒自動存至localStorage/SQLite

## eGUI State Machine

```
草稿 → 已送出 → 已開立
           ↓
        已作廢

狀態轉換說明:
- 草稿: 本地建立，尚未送出
- 已送出: 已提交至eGUI系統， awaiting驗證
- 已開立: eGUI驗證通過，發票正式開立
- 已作廢: 發票已作廢（需填寫作廢原因）
```

## Validation Rules

- 統一編號：必填，8碼+MOD11驗證
- 品項：至少一行，數量>0，單價>=0
- 日期：不可晚於今日
- 總計：系統自動計算，不可編輯
