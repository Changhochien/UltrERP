## Story

As a frontend developer,
I want a charting library installed in the frontend,
So that the owner dashboard can render revenue trend lines, bar charts, and pie charts.

## Acceptance Criteria

1. [AC-1] **Given** the frontend project at `src/`
   **When** the developer runs `pnpm add recharts`
   **Then** `recharts` is listed in `package.json` dependencies
   **And** the import `import { LineChart, Line, BarChart, Bar, PieChart, Pie, Cell, XAxis, YAxis, CartesianGrid, Tooltip, Legend, ResponsiveContainer } from 'recharts'` works in TypeScript without type errors

2. [AC-2] **Given** recharts is installed
   **When** a developer imports any recharts component in a `.tsx` file under `src/`
   **Then** Vite dev server starts without errors
   **And** `pnpm build` completes with exit code 0 (no TS errors from recharts types)

3. [AC-3] **Given** recharts is installed
   **When** the existing dashboard components at `src/domain/dashboard/components/` are inspected
   **Then** no existing component imports or uses a different charting library that would conflict
